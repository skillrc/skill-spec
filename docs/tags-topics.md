# Tags and Topics Guidance

## Intent

`tags` and `topics` work together, but they are not the same thing.

## Tags

Tags are narrow keywords.

They should answer:

> What concrete terms would someone search for to find this skill?

Examples:

- `git`
- `commits`
- `cleanup`
- `playwright`
- `postgres`

## Topics

Topics are broader themes.

They should answer:

> What thematic area does this skill belong to?

Examples:

- `version-control`
- `code-hygiene`
- `repository-maintenance`
- `frontend-development`
- `technical-research`

## Heuristic

- Use tags for concrete search terms.
- Use topics for grouping and navigation.
- Prefer 3-5 tags and 1-3 topics when possible.
- Avoid repeating the exact same concept in both fields unless it is truly necessary.

## Recommended workflow

1. Generator suggests tags/topics.
2. Author reviews the suggestions.
3. Author accepts, edits, or overrides.
4. Final SKILL file keeps both fields present.
